# دليل مستخدم منصّة سيجما PMO (النسخة العربية)

> نظام تشغيل ذكي للاستثمار والتسليم والحوكمة في قطاع الإنشاءات.
> يغطّي هذا الدليل كل صفحة في المنصّة، وما تفعله كل ميزة وأداة، وكيفية استخدامها بالضبط —
> بما في ذلك ميزة بريمافيرا وميزة نماذج المعلومات (BIM)، وما الذي يحدث عند ربط واجهة Claude API.

---

## 1. عن المنصّة

سيجما PMO هي **نظام دعم قرار للحوكمة**، وليست أداة تقارير. تستوعب بيانات المشروع (جداول
Primavera P6، وملفات Excel/CSV، ونماذج BIM/IFC، والتقارير اليومية)، وتوحّدها في نموذج مرجعي
واحد، وتُشغّل محرّكات حوكمة حتمية عبر دورة حياة استثمار وتسليم من 17 مرحلة، وتعرض حُكماً موحّداً
(أخضر / أصفر / برتقالي / أحمر) لكل عقدة — قابلاً دائماً للتتبّع رجوعاً إلى الملف المصدر الذي جاء منه.

تنتظم المنصّة في أربع مجموعات تنقّل تظهر في الشريط الجانبي الأيسر:

1. **قيادة الحوكمة** — الوجهات: مركز القيادة، والرؤية التنفيذية، والهيكل الهرمي
   (مؤسسة ← محفظة ← برنامج ← مشروع)، وسجل الوكلاء، وشاشات دورة حياة الاستثمار والتسليم
   (الفرص، الجدوى، حصر الكميات، المشتريات، الإيرادات، التمويل، التنبؤ، القابلية للتمويل،
   السلامة، الحريق وسلامة الأرواح، الجهات، المرافق، الجاهزية التشغيلية).
2. **طبقات الوكلاء (L0–L8)** — العمود الفقري التحليلي: المعرفة (L0)، إدخال البيانات (L1)،
   المراجعة/القواعد (L2)، القرارات (L3)، التحليلات/القيمة المكتسبة (L4)، المخاطر (L5)،
   المطالبات (L6)، التنفيذي (L7)، حوكمة سيجما (L8).
3. **الأدوات والأدلة** — خطوط الأساس، المحاكاة، التضاربات، المخططات، الخطابات، المصادر،
   التقارير، المستودع، الأدلة، الاعتماد، المقارنة.
4. **الإدارة** — الأدوار، المستخدمون، إعداد الحوكمة، السياسات، الشخصيات، الإعدادات، التدقيق.

### مفاهيم متكرّرة (اقرأها مرّة، وتنطبق في كل مكان)

- **الأولوية للحتمية.** كل رقم يُحسَب مباشرةً من بيانات مشروعك بصيغة مُسمّاة — لا يُخمّنه ذكاء
  اصطناعي. ودور الذكاء الاصطناعي هو *السرد* فقط.
- **سرد بالذكاء الاصطناعي مع استشهادات.** حيث تظهر لوحة **تحليل بالذكاء الاصطناعي**، تشرح
  النتائج المحسوبة بلغة واضحة وتُسنِدها بوسوم `[SOURCE: id]`. (غير فعّالة حتى تربط Claude API —
  راجع فصل التكاملات.)
- **بوّابة الاعتماد البشري.** مسودّات الذكاء الاصطناعي (خطابات، قرارات) لا تُرسَل تلقائياً —
  يعتمدها إنسان أولاً. وتتطلّب القرارات الحرجة وخطوط الأساس معتمِدَين *مختلفَين*.
- **مُبدّل المشروع.** عنصر تحكّم أعلى التطبيق يربط معظم الصفحات بمشروع واحد. إذا بدت الأرقام
  خاطئة، فتحقّق من المشروع المختار أولاً.
- **وصول مقيّد بالصلاحيات.** يرى كلٌّ من الأدوار الخمسة عشر شريطاً جانبياً مختلفاً. والأزرار التي
  لا يتيحها دورك تظهر مُعطَّلة أو مخفية. وتذكر ملاحظات الوصول في هذا الدليل الصلاحية التي تحكم كل صفحة.
- **ثنائي اللغة + RTL.** كل شاشة تبدّل بين الإنجليزية والعربية (أعلى اليمين)، مع تخطيط كامل من
  اليمين إلى اليسار بالعربية.

### البدء

1. افتح `/auth`، وانقر شارة دورك (أو اكتب بريدك + كلمة مرورك)، وسجّل الدخول.
2. تأكّد من **المشروع** في المُبدّل أعلى الصفحة.
3. تتبع معظم صفحات الحوكمة نمطاً واحداً: **أضِف السجلّات ← اضغط *تشغيل الحوكمة* ← اقرأ المؤشّر
   والنتائج ← افتح لوحة *تحليل الذكاء الاصطناعي*** للسرد.
4. اضبط اللغة والسمة من عناصر التحكّم أعلى اليمين.

## 2. التكاملات — Claude API وPrimavera P6 وAutodesk BIM

تُهيَّأ التكاملات الثلاثة في مكان واحد: **الإدارة ← إعدادات المنصّة** (`/admin/settings`). ويُشفَّر
كل اعتماد **عند التخزين (AES-256-GCM)** ولا يُعرَض مجدّداً — فتعرض الشاشة بصمةً وسجلّ تدقيق فقط.
ويسري ضبط أو تغيير أي اعتماد **خلال ثوانٍ، دون إعادة تشغيل**. ولا يُشترط أيٌّ من الثلاثة لتشغيل
المنصّة؛ بل يُضيف كلٌّ منها قدرةً جديدة.

### 2.1 ربط Claude API — ماذا يحدث

**ما هو.** Claude هو النموذج اللغوي الكبير الذي يكتب السرود، ويصوغ خطابات FIDIC، ويقترح حلول
التضاربات، ويقرأ المخططات والمستندات الممسوحة، ويشرح النتائج بلغة واضحة في أنحاء المنصّة.

**كيفية الربط.** `/admin/settings` ← **مفتاح Anthropic API** ← الصق مفتاحك `sk-ant-…` من
console.anthropic.com ← احفظ. (اختيارياً، اضبط فئة نموذج لكل وكيل من الإدارة ← مركز إعداد الحوكمة.)

**في اللحظة التي يُحفَظ فيها مفتاح صالح، "تُضيء" المنصّة:**

- يتحوّل شريط الإعدادات من **"Claude معطّل"** (كهرماني) إلى **"Claude مفعّل"** (أخضر)، مُظهِراً
  مصدر المفتاح والنموذج/الفئة الافتراضية.
- كل سطح كان يعمل بالحتمية فقط يبدأ بإنتاج تحليل مكتوب:
  - **لوحات تحليل الذكاء الاصطناعي** على كل طبقة حوكمة (السلامة، الجهات، حصر الكميات، التمويل،
    التنبؤ…) تسرد النتائج المحسوبة.
  - **التقارير الشهرية/الدورية** (`/reports/monthly`) تكتسب سرداً تنفيذياً كاملاً بالعربية
    والإنجليزية بدل النقاط المجرّدة.
  - **خطابات FIDIC** (`/letters`) تُصاغ ثنائية اللغة عبر الشخصية الخبيرة `fidic-redbook-expert`
    مع استشهادات إلزامية.
  - **حلّال التضاربات** (`/clashes`) يقترح ثلاثة خيارات حلٍّ لكل تضارب.
  - **إدخال الرسوم المفاهيمية والمسح الضوئي** (`/feasibility`, `/repository`) يستخرج البيانات من
    المخططات والمستندات عبر Claude Vision.
  - وتُفعَّل أيضاً **مخطِّط خط الأساس** و**مراجعة الهيكل التنظيمي** و**مقارنة الذكاء الاصطناعي بالبشري**.
- **ما لا يتغيّر (عقد الأمان):**
  - **الأولوية للحتمية** — لا يزال المحرّك يحسب كل رقم؛ ويسرده Claude فقط. ولا يخترع رقماً أبداً.
  - **استشهادات `[SOURCE: id]`** — كل ادّعاء مهني يجب أن يستشهد ببندٍ في سجل المصادر؛ وأي استشهاد
    مفقود أو ملفّق يُرفَض ويُبقى على البديل الحتمي.
  - **بوّابة الاعتماد البشري** — لا تُرسَل مسودّات الذكاء الاصطناعي تلقائياً؛ ولا يوجد زر "إرسال".
  - **إعادة التحميل التلقائي** — غيّر المفتاح أو احذفه فتلتقطه المنصّة خلال ثوانٍ. وعند حذفه يعود كل
    سطح بسلاسة إلى المخرجات الحتمية فقط.
- **ضبط التكلفة.** تُحدَّد فئة نموذج كل وكيل (Haiku / Sonnet / Opus / الافتراضي) لكل وكيل على حدة في
  الإدارة ← مركز إعداد الحوكمة، فتصرف على Opus حيث يهمّ الحُكم فقط. والفوترة بحسب الاستخدام على
  console.anthropic.com.

> باختصار: **بدون** المفتاح تكون المنصّة محرّك حوكمة حتمياً عاملاً بالكامل؛ **ومع** المفتاح تكتسب
> راوياً وصائغاً — وحساب الحوكمة متطابق في الحالتين.

### 2.2 Primavera P6 — استيراد الملفات + سحب حيّ عبر REST

تدعم سيجما بريمافيرا P6 بطريقتين.

**أ) رفع الملفات (متاح دائماً، دون اعتماد).** في `/input` (أو عبر webhook الخاص بـ P6)، ارفع تصدير
P6 — `.xer`، أو PMXML `.xml`، أو جدول أنشطة P6 بصيغة `.pdf`. يُحلَّل إلى النموذج المرجعي (مشاريع،
أنشطة، موارد، تخصيصات)، ويُتحقَّق منه، ويُمنَح درجة ثقة، ويُؤرشَف بصورة غير قابلة للتعديل.

**ب) سحب حيّ عبر EPPM REST (جديد — يحتاج اعتماداً).** اتصل مباشرةً بخادم Primavera P6 EPPM لديك
واسحب مشروعاً عند الطلب، دون تصدير يدوي.

1. هيّئ في `/admin/settings` ← **Primavera P6**: رابط EPPM REST
   (مثل `https://host/p6ws/restapi`)، واسم قاعدة البيانات، واسم المستخدم، وكلمة المرور (يُفضَّل حساب
   خدمة بصلاحية قراءة فقط).
2. `GET /integrations/p6/status` يؤكّد الاتصال (أضِف `?probe=true` لاختبار الاعتماد حيّاً).
3. `GET /integrations/p6/projects` يسرد المشاريع التي يراها اعتمادك.
4. `POST /integrations/p6/sync {projectId}` يسحب ذلك المشروع حيّاً.

يمرّ السحب الحيّ عبر **نفس** مسار التحقّق ← التوحيد ← الثقة ← التدقيق كرفع الملف، فيستقرّ في النموذج
المرجعي بصورة مطابقة لملف `.xer` مرفوع. عندها تستهلكه كل الطبقات اللاحقة (التحليلات/القيمة المكتسبة،
خطوط الأساس، المخاطر، المطالبات، التنبؤ) بالطريقة نفسها.

### 2.3 Autodesk BIM (APS) — IFC محلّي + تحويل نموذج حيّ ("pim")

يربط تكامل BIM المنصّةَ بنماذجك ثلاثية الأبعاد كي تُحكَم الكميات والتضاربات. (يُكتَب أحياناً "pim".)

**أ) رفع IFC محلّي (متاح دائماً، دون اعتماد).** في `/drawings` ← **نماذج BIM**، ارفع ملف `.ifc`
بصيغة STEP (≤ 50 ميغابايت). تحلّل المنصّة أعداد العناصر (الجدران، البلاطات، الأعمدة، الكمرات،
الأبواب، النوافذ، الفراغات، الطوابق)، وتتحقّق من النموذج، وتخزّنه كسجلّ `bim-model` — الذي يحوّله
مسار حصر الكميات إلى حصر ← تكلفة مصنّفة ← نتائج حوكمة.

**ب) تحويل حيّ عبر Autodesk APS (جديد — يحتاج اعتماداً).** اربط تطبيق Autodesk Platform Services
لتحويل نماذج Revit / IFC / Navisworks الأصلية في السحابة وقراءة خصائص عناصرها.

1. هيّئ في `/admin/settings` ← **Autodesk APS**: معرّف العميل + السرّ (أنشئ تطبيقاً مجانياً من
   aps.autodesk.com/myapps).
2. `GET /integrations/autodesk/status` (أضِف `?probe=true` لاختبار الاعتماد بجلب رمز وصول).
3. `POST /integrations/autodesk/import {projectKey, filename, contentBase64}` يرفع النموذج إلى
   Autodesk، ويُجري تحويل Model Derivative، وينتظر حتى الاكتمال، ويقرأ خصائص العناصر، ويربطها بنفس
   أعداد عناصر BIM، ويكتب سجلّ `bim-model`.
4. تُوفَّر نقطة رمز عارض (`viewables:read`) لعارض ثلاثي الأبعاد داخل المتصفّح.

فالنموذج المُحوَّل عبر Autodesk يغذّي **نفس مسار BIM ← الكمية ← التكلفة ← الحوكمة** تماماً كملف IFC
محلّي. ثم تنتقل الكميات المُستخرَجة إلى حصر الكميات (BIM مقابل BOQ) والمشتريات (BIM مقابل المُشترى
مقابل المُركّب).

> **ملاحظة أمانة.** الكميات المشتقة من BIM *إرشادية* (أعداد العناصر × أحجام اسمية موثّقة)، ومُعلَّمة
> بوضوح بأنها مشتقة. وهي تدعوك لتأكيد BOQ المقيس، ولا تحلّ محلّه.

## 3. الحوكمة ومركز القيادة

---

### لوحة القيادة الرئيسية — `/`

**الوصول:** أي مستخدم مُسجَّل الدخول (بوّابة مصادقة أساسية).

**الغرض منها:** شاشة الدخول للمنصّة والفحص اليومي للحالة. تمنح صورة فورية للمشروع المُحدَّد — مؤشرات أدائه التنفيذية، وحجم البيانات المُستوعَبة، وعدد التنبيهات المفتوحة، وآخر ملخّص مُولَّد بالذكاء الاصطناعي.

**أهم الميزات والأدوات:**
- **شريط المؤشرات التنفيذية** — ست بطاقات: صحة المشروع (/100)، ثقة الحوكمة (/100)، التأخير المتوقع (أيام)، تجاوز التكلفة (%)، تاريخ الإنجاز المتوقع، وصحة المحفظة (/100).
- **أربع بطاقات إحصائية** — عمليات الاستيعاب، إجمالي التنبيهات، الحرجة، التحذيرات (كل بطاقة تنقلك لصفحتها).
- **رسوم تحليلية** — مقياس ثقة، ومخطط للتنبيهات حسب الخطورة، ومخطط لعمليات الاستيعاب، وتوزيع المحلّلات.
- **بطاقة آخر استيعاب** و**بطاقة آخر ملخّص** (مع وسم المصدر ونسبة الثقة).

**طريقة الاستخدام:** سجّل الدخول؛ تأكّد من المشروع في المُبدّل؛ اقرأ شريط المؤشرات؛ انقر أي بطاقة للانتقال لصفحة التفاصيل.

**المُدخلات/المُخرجات:** تجلب عمليات الاستيعاب والتنبيهات والمؤشرات وآخر ملخّص. للقراءة فقط.

**نصيحة:** إذا بدت الأرقام خاطئة، فتحقّق من مُبدّل المشاريع أولاً — فالتنبيهات والملخّصات مرتبطة بالمشروع.

---

### مركز قيادة الحوكمة — `/governance-command`

**الوصول:** يتطلّب صلاحية `canEvaluateRules`.

**الغرض منها:** مركز دعم قرار الحوكمة في الطبقة 8 — المرجعية النهائية. يعرض لكل عقدة الحُكم الموحّد رباعي التصنيف، والوكلاء الذين أنتجوه، والمخاطر والمطالبات والإجراءات المفتوحة خلفه. وهو يُعيد احتساب الحالة ويُصدر الإجراءات من جديد.

**أهم الميزات والأدوات:**
- **مخطط توزيع حالة الحوكمة** و**قائمة الإشراف على المحفظة** (كل عقدة صفّ قابل للاختيار).
- **بطاقة الحُكم الموحّد** (شارة الحالة، درجة المخاطر، الوكلاء، وزر **إعادة الاحتساب (الطبقة 8)**).
- **بطاقة الإجراءات التصحيحية** (بدء / تحديد كمنجز / تجاهل)، و**الإجراءات المقترحة**، و**مسارات التصعيد**، و**تحليل الأثر التنفيذي**، و**أثر تحقيق المنافع**.

**طريقة الاستخدام:** راجع المخطط ← اختر عقدة ← اقرأ حُكمها ← **إعادة الاحتساب (الطبقة 8)** ← عالِج قائمة الإجراءات.

**المُدخلات/المُخرجات:** تقرأ نظرة التوحيد والإجراءات والتصعيدات؛ وتكتب إعادة الاحتساب وتغييرات حالة الإجراءات (التوحيد يعمل بالسحب وقابل للتكرار).

**نصيحة:** إذا أظهرت العقدة "لم تُشغَّل أي وكلاء بعد"، فشغّل سلسلة الوكلاء أولاً (صفحة الوكلاء).

---

### لوحة القيادة التنفيذية — `/executive`

**الوصول:** يتطلّب `canEvaluateRules`؛ وبطاقة درجات المؤسسة تتطلّب صلاحية قراءة الكل.

**الغرض منها:** رؤية الذكاء التنفيذي في الطبقة 7. تُجمّع مؤشرات الأداء الاستراتيجية من كل طبقة للمشروع المُحدَّد، إضافةً إلى بطاقة درجات حوكمة على مستوى المؤسسة.

**أهم الميزات والأدوات:**
- **بطاقة الدرجات** (درجة مركّبة وستة مقاييس)، و**عنوان الحوكمة**، و**بطاقات المؤشرات** (الحوكمة، صحة الجدول والتكلفة، SPI، CPI، التعرّض للمخاطر، المطالبات، الإجراءات)، و**القسم الاستراتيجي** (مواءمة الأهداف، تتبّع قيمة المحفظة، تحقيق المنافع).

**طريقة الاستخدام:** اختر المشروع ← اقرأ بطاقة الدرجات والعنوان ← تصفّح المؤشرات ← استعن بالبطاقات الاستراتيجية ← **تحديث**.

**المُدخلات/المُخرجات:** تقرأ النظرة التنفيذية والمؤشرات والدرجات. للقراءة فقط.

**نصيحة:** معظم الأرقام قواعد حتمية ذات "أساس" مُعلَن — اقرأ ملاحظة الأساس أسفل كل مقياس قبل التصرّف.

### الهيكل الهرمي للحوكمة — `/hierarchy`

**الوصول:** يتطلّب العرض `canEvaluateRules`؛ والإنشاء/الإرفاق/ضبط المرحلة/إعادة الاحتساب يتطلّب `canManageHierarchy` (مسؤول، عميل، مالك، PMO).

**الغرض منها:** بنية الحوكمة متعدّدة المستويات: مؤسسة ← محفظة ← برنامج ← مشروع. وتحمل كل عقدة حالتها المُجمَّعة (أسوأ حالة بين العُقد الفرعية، مُرجَّحة بالـ BAC).

**أهم الميزات والأدوات:** مفتاح دلالات الحالة، ووسوم المؤشرات لكل عقدة (CPI، SPI، R، C، B)، وشجرة الحوكمة، وأزرار الإنشاء (للمدير)، ولوحة العقدة (مع **إعادة احتساب الحالة** وشريط مرحلة دورة الحياة)، ونموذج إرفاق مشروع.

**طريقة الاستخدام:** انقر عقدة ← (للمدير) أنشئ البنية، أرفِق مشروعاً، اضبط المرحلة ← **إعادة احتساب حالة الحوكمة**.

**المُدخلات/المُخرجات:** تقرأ الشجرة والمؤشرات؛ وتكتب العُقد والارتباطات وتغييرات المرحلة وتُطلق إعادة الاحتساب.

**نصيحة:** عرّف العُقد بمفتاحها التجاري (ENTERPRISE-001، P-1000) لا باسم العرض.

---

### سجل الوكلاء (L0–L8) — `/agents`

**الوصول:** يتطلّب `canEvaluateRules`؛ وأزرار التشغيل تظهر لتلك الأدوار فقط.

**الغرض منها:** سجل كل وكلاء الحوكمة عبر الطبقات 0–8. يُثبت العقد المعياري للوكيل — كل وكيل في بطاقة موحّدة (الهدف، المدخلات، المعالجة، المخرجات، الثقة، التصعيد، التدقيق) مع تشغيله بنقرة واحدة.

**أهم الميزات والأدوات:** جدول صحة الوكلاء (سليم/متدهور/متعثّر)، وبطاقات العقد (مع فئة النموذج وحالة آخر تشغيل)، وزر **تشغيل `<الوكيل>`** و**تشغيل سلسلة الوكلاء الكاملة** (L1→L8).

**طريقة الاستخدام:** تأكّد من المشروع ← راجع جدول الصحة ← شغّل وكيلاً واحداً أو السلسلة كاملة ← **تحديث**.

**المُدخلات/المُخرجات:** تقرأ قائمة الوكلاء وتشغيلاتهم وتقرير الصحة؛ وتكتب عبر التشغيل (الوكلاء المُعطّلون يرفضون التشغيل بـ409).

**نصيحة:** الوكيل "المُعطّل" زرّه مقفول — أعد تفعيله من الإدارة ← مركز إعداد الحوكمة.

---

### المشاريع — `/projects`

**الوصول:** أي مستخدم مُسجَّل الدخول.

**الغرض منها:** القائمة الرئيسية لكل المشاريع مع حزمة درجاتها الحتمية وأعدادها — لترتيب المشاريع ومقارنتها حسب الحوكمة والمخاطر والاستثمار والدرجة المركّبة.

**أهم الميزات والأدوات:** أربع بطاقات إحصائية، وجدول قابل للبحث والفرز (الاسم، المفتاح، الحالة، الدرجة المركّبة مع وسوم الترتيب، الحوكمة، المخاطر، الاستثمار، التنبيهات، التشغيلات، الثقة)، ووسوم درجات ملوّنة (المخاطر معكوسة).

**طريقة الاستخدام:** الجدول مفروز حسب الدرجة المركّبة ← ابحث ← أعد الفرز ← افرز حسب التنبيهات/الثقة للفرز.

**المُدخلات/المُخرجات:** تقرأ المشاريع والتنبيهات والاستيعاب مربوطةً بالمفتاح التجاري (لا بالمعرّف). للقراءة فقط.

**نصيحة:** ثِق بالدرجة المركّبة وتحقّق بعمود المخاطر (100 = الأسوأ).

---

## 4. طبقات التحليل (L0–L6)

تُشكّل هذه الطبقات السبع العمود الفقري الذكي. وهي تتكامل بعضها فوق بعض: المعرفة تُغذّي القواعد، والقواعد تقرأ البيانات المُدخَلة، والقرارات تستند إلى القواعد، وتستنبط التحليلات والمخاطر والمطالبات من البنود المرجعية نفسها. و**«حتمي»** يعني أنّ الأرقام تُحسَب مباشرةً من بيانات مشروعك، لا أن يُخمّنها نموذج.

---

### محرّك المعرفة والقواعد — `/knowledge`

**الوصول:** أي مستخدم يقرأ؛ وزر «تسجيل درس» للمحرّرين (مسؤول، عميل، مالك).

**الغرض منها:** الطبقة 0، الأساس الذي تستند إليه كل الطبقات. تضمّ مكتبة قواعد سيجما، وسجل المعايير (FIDIC، PMI/PMBOK، ISO، AACE، Primavera)، وأطر الحوكمة، والدروس المستفادة، والمقاييس المرجعية للتكاليف والعوائد.

**أهم الميزات والأدوات:** شريط بحث موحّد، وخمس علامات تبويب (مكتبة القواعد، المعايير، الأطر، الدروس، المقاييس)، وبطاقات قواعد، وتبويب مقاييس مرجعية، ونموذج تسجيل درس.

**طريقة الاستخدام:** ابحث بمصطلح ← أو تصفّح تبويباً (ابدأ بمكتبة القواعد) ← افتح المقاييس لمراجعة افتراضات الجدوى ← (للمحرّر) سجّل درساً.

**المُدخلات/المُخرجات:** تقرأ قاعدة المعرفة؛ وتُخرِج دروساً مستفادة تستند إليها الطبقات اللاحقة.

**نصيحة:** ابحث بالرمز الدقيق أو كلمة مميّزة للحصول على أنظف النتائج.

---

### إدخال البيانات والاستيراد — `/input`

**الوصول:** الأدوار المخوَّلة باستيراد الجداول (مسؤول، مقاول، PMO).

**الغرض منها:** الطبقة 1، حيث تدخل الجداول والتقارير. ترفع ملف P6 أو MS Project أو Excel أو CSV؛ فيُؤرشَف، ويُحلَّل إلى بنود أنشطة مرجعية، ويُمنَح درجة ثقة.

**أهم الميزات والأدوات:** منطقة رفع (`.xer`، `.xml`، `.xlsx`، `.csv`، `.pdf`، ≤24 ميغابايت) + **إدخال**، ولافتة نتيجة، وجدول **عمليات الإدخال الأخيرة** مع شريط ثقة.

**طريقة الاستخدام:** اسحب الملف (أو **تصفّح**) ← تأكّد من الاسم/الحجم ← **إدخال** ← راجع النتيجة والصف الجديد.

**المُدخلات/المُخرجات:** ملفات خام داخلة؛ بنود مرجعية وسجل مصدر غير قابل للتعديل ودرجة ثقة خارجة.

**نصيحة:** راقب شريط الثقة — والدرجة المنخفضة تعني مصدراً شحيحاً. (راجع أيضاً التكاملات ← Primavera P6 للسحب الحيّ بدل رفع الملفات.)

---

### تقييم القواعد والإنذارات (المراجعة) — `/review`

**الوصول:** الأدوار القادرة على تقييم القواعد؛ مُقيَّد بالمشروع المختار.

**الغرض منها:** الطبقة 2، محرّك القواعد. يفحص البيانات المُدخَلة في ضوء مكتبة القواعد، ويُظهر الإنذارات حسب الخطورة، ويحوّلها إلى قرارات حوكمة، ويُنتج ملخّصاً أسبوعياً.

**أهم الميزات والأدوات:** شرائح خطورة، و**تقييم · {المشروع}**، و**تشغيل سير عمل الحوكمة**، و**تشغيل · كل المشاريع**، و**الملخّص الأسبوعي**، وبطاقات قرارات.

**طريقة الاستخدام:** اختر المشروع ← **تقييم** ← صفِّ على **الحرج** ← اقرأ بطاقات القرار ← **الملخّص الأسبوعي** عند الحاجة.

**المُدخلات/المُخرجات:** تقرأ البنود المرجعية (L1) والقواعد (L0)؛ وتُخرِج إنذارات وقرارات (تستهلكها `/decisions`) وملخّصاً.

**نصيحة:** استخدم **تشغيل سير عمل الحوكمة** لتوليد الإنذارات والقرارات في خطوة واحدة.

---

### قرارات الحوكمة وربط FIDIC — `/decisions`

**الوصول:** أي مستخدم؛ مُقيَّد بالمشروع الحالي.

**الغرض منها:** الطبقة 3، سجل الحوكمة. يُربط كل قرار بنموذج وببند FIDIC، مع الطرف المسؤول ومستوى التصعيد وحالة المراجعة، ومسار إثبات كامل.

**أهم الميزات والأدوات:** شرائح حالة، وجدول قابل للبحث والفرز (الوقت، الخطورة، الرمز، النموذج، الطرف، بند FIDIC، التصعيد، الحالة، **إثبات**). ويفتح **إثبات** المسار: القرار ← الإنذار ← تقييم القاعدة ← الاستيراد ← الملف المصدر (SHA-256) ← الثقة.

**طريقة الاستخدام:** اختر المشروع ← صفِّ (مثل **قيد الانتظار**) ← ابحث ← انقر **إثبات** لتوسيع السلسلة.

**المُدخلات/المُخرجات:** تقرأ القرارات والإنذارات (L2) وسلسلة الإثبات إلى ملفات L1؛ وتُخرِج مسار أدلة جاهزاً للتدقيق.

**نصيحة:** قبل التصعيد، افتح **مسار الإثبات** — فاسم الملف وبصمته وثقته يجيبون "من أين جاء هذا الرقم؟".

---

### التحليلات والقيمة المكتسبة — `/analytics`

**الوصول:** الأدوار القادرة على تقييم القواعد؛ عرض للمشروع وآخر للمحفظة.

**الغرض منها:** الطبقة 4، محرّك التحليلات. يحسب القيمة المكتسبة (SPI، CPI، EAC، VAC)، والجدول المكتسب، والإنتاجية، والاتجاهات، وتجميع المحفظة — حتميّاً.

**أهم الميزات والأدوات:** تبويبا «هذا المشروع»/«المحفظة»، ولافتة تنبؤ، وبطاقة الجدول المكتسب، ومقياسا SPI وCPI، ورسم اتجاهات، وبلاطات القيمة المكتسبة، ولوحة الإنتاجية.

**طريقة الاستخدام:** اقرأ لافتة التنبؤ ← تحقّق من المقاييس (≥1.0 جيد) ← افتح الجدول المكتسب ← راجع الاتجاهات ← انتقل إلى **المحفظة**.

**المُدخلات/المُخرجات:** تقرأ بنود الأنشطة (مع التكاليف)؛ وتُخرِج مؤشرات القيمة المكتسبة والتنبؤ ومقاييس المحفظة — وتُغذّي المخاطر.

**نصيحة:** يلزم الاتجاهَ لقطتان على الأقل — أعد تشغيل التحليلات أسبوعياً.

---

### سجل المخاطر — `/risk`

**الوصول:** الأدوار القادرة على تقييم القواعد؛ **تشغيل وكيل المخاطر** لها.

**الغرض منها:** الطبقة 5، سجل المخاطر. تُشتقّ حتميّاً من إنذارات L2 وقيمة L4، وتُقيَّم وفق الاحتمالية × الأثر، وتُطابَق مع خيارات معالجة، مع تجميع لمخاطر المحفظة.

**أهم الميزات والأدوات:** شريط مخاطر المحفظة، وخريطة حرارية 5×5، ومصفوفة ارتباط، وبطاقات مخاطر مُقيَّمة (مع مُحفّز تصعيد وخيارات معالجة)، و**تشغيل وكيل المخاطر**.

**طريقة الاستخدام:** **تشغيل وكيل المخاطر** ← اقرأ الخريطة (أعلى يمين = الأخطر) ← اعمل بالبطاقات حسب الأولوية ← وسّع خيارات المعالجة.

**المُدخلات/المُخرجات:** تقرأ الإنذارات (L2) والقيمة المكتسبة (L4)؛ وتُخرِج سجل مخاطر مُقيَّماً ومعالَجاً.

**نصيحة:** مُحفّز التصعيد يخبرك بالشرط الدقيق للرفع — فاضبط تقاريرك حوله.

---

### سجل المطالبات — `/claims`

**الوصول:** الأدوار القادرة على تقييم القواعد؛ **تشغيل وكيل المطالبات** لها.

**الغرض منها:** الطبقة 6، سجل المطالبات والنزاعات. تُحدَّد المطالبات (تمديد مدّة، تكلفة، أمر تغييري، تعطيل) حتميّاً من أحداث التأخير والقرارات، مع بند FIDIC وفحص استحقاق ودرجة جاهزية وحزمة قابلة للطباعة مدعومة بالأدلة.

**أهم الميزات والأدوات:** بطاقات مطالبات، وشريط **جاهزية المطالبة** (أدلة/استحقاق/قيمة مالية/سرد)، ومعايير استحقاق، و**عرض حزمة المطالبة** ← **طباعة/PDF**، ورابط **صياغة خطاب FIDIC**.

**طريقة الاستخدام:** **تشغيل وكيل المطالبات** ← رتّب حسب النوع والبند والاستحقاق ← اقرأ شريط الجاهزية ← وسّع المعايير ← **عرض الحزمة** ← **طباعة/PDF**.

**المُدخلات/المُخرجات:** تقرأ أحداث التأخير والقرارات (L2–L3)؛ وتُخرِج مطالبات مُقيَّمة وحزمة قابلة للطباعة؛ وتُحيل للخطابات.

**نصيحة:** لا تقدّم مطالبةً قبل أن يصبح شريط جاهزيتها أخضر — فالتفصيل يدلّك على الركيزة التي تُعزَّز أولاً.

## 5. الاستثمار والكميات والتجارة

تنقل هذه الشاشات الثماني المشروع من فكرة من سطر واحد حتى حزمة تمويل جاهزة للمُقرِض. وكلٌّ منها مرتبط بالمشروع المختار، ويعمل بمنطق حتمي، ويحمل عادةً لوحة **تحليل بالذكاء الاصطناعي** تشرح الأرقام.

---

### استخبارات الفرص الاستثمارية — `/opportunity`

**الوصول:** `canRunOpportunity`.

**الغرض منها:** البوابة الأولى لدورة حياة الاستثمار. تُقيَّم الفكرة من 0 إلى 100 عبر جاذبية السوق والمنافسة والتمويل والتعقيد التنظيمي، لتنتهي بتوصية *المضي/المراقبة/الرفض* — قبل صرف أي جهد على الجدوى.

**أهم الميزات والأدوات:** نموذج تقييم فرصة جديدة (يُشغّل وكيل `ext.opportunity`)، وقائمة فرص مُقيَّمة (الدرجة، التوصية، أربع بطاقات فرعية)، وبطاقة استخبارات السوق (الطلب/العرض/المنافسة + معايير القطاع).

**طريقة الاستخدام:** املأ النموذج ← **تقييم الفرصة** ← اقرأ الدرجة والتوصية ← افحص البطاقات الأربع ← انقل فكرة «المضي» للجدوى.

**المُدخلات/المُخرجات:** حقول عالية المستوى داخلة؛ سجل تقييم محفوظ قابل للتدقيق خارج (مدخل الجدوى).

**نصيحة:** اضبط محدّدات بطاقة السوق أولاً — فهي تُعبّئ نوع المشروع والمدينة والدولة مسبقاً.

---

### الاستثمار ودراسة الجدوى — `/feasibility` و`/feasibility/[id]`

**الوصول:** `canRunFeasibility`.

**الغرض منها:** تحويل فكرة (أو رسم مفاهيمي) إلى تقييم استثماري سريع (المستوى 1) ودراسة احترافية قابلة للتمويل (المستوى 2): NPV، IRR، فترة الاسترداد، DSCR، تصنيف المخاطر، وتوصية حوكمة.

**أهم الميزات والأدوات (أربعة تبويبات):**
- **المستوى 1 · التقييم** — لافتة توصية، وخمس بطاقات مؤشرات، ورسم CAPEX، وجدول تدفقات نقدية سنة بسنة؛ و**تشغيل التقييم السريع**.
- **المستوى 2 · الدراسة** — **توليد الدراسة الاحترافية** (17 بنداً، كل بند قابل للتوسعة ومُعتمَد عبر **اعتماد البند**).
- **الحزم** — تُجمِّع البنود المُعتمَدة في حزم (مستثمر/شريك/بنك) مع **طباعة/حفظ PDF**.
- **الرسوم المفاهيمية** — ارفع رسماً/PDF؛ يقترح الاستخراج المُدخلات؛ وتُراجِع وتعتمد قبل التطبيق.

**طريقة الاستخدام:** **فرصة جديدة** ← **تشغيل التقييم السريع** ← (اختياري) ارفع رسماً واستخرج وطبّق ← **توليد الدراسة** ← اعتمد البنود ← **الحزم** ← **طباعة/PDF**.

**المُدخلات/المُخرجات:** مُدخلات الفرصة (والرسم) داخلة؛ نموذج التقييم والدراسة والحزم خارجة. وتُغذّي مخرجات الدين/DSCR/CAPEX شاشتي التمويل والقابلية للتمويل.

**نصيحة:** الاعتماد هو البوابة البشرية — الحزم تُجمّع البنود *المُعتمَدة*، فاعتمدها قبل التصدير.

---

### حصر الكميات وحوكمة التكلفة — `/quantity-survey` (ميزة BIM بتفصيل موسّع)

**الوصول:** `canRunQuantitySurvey`.

**الغرض منها:** حوكمة كاملة للكميات والتكلفة من المفهوم حتى الحساب الختامي، مصنّفة وفق أربعة معايير عالمية (NRM / UniFormat / MasterFormat / CESMM). هنا تحوّل المنصّة النموذج ثلاثي الأبعاد إلى تكلفة قابلة للحوكمة: **BIM ← الكمية ← التكلفة ← الحوكمة**. وهو محرّك تصنيف من تطوير سيجما — لا قاعدة بيانات تكلفة تجارية مُسعَّرة خلفه؛ فكل رقم حساب حتمي على معيار جدوى سيجما.

**مسار BIM ← الكمية ← التكلفة ← الحوكمة (ما يحدث فعلياً):**
1. **النموذج (BIM).** عند رفع نموذج IFC (المخططات ← نماذج BIM، أو عبر موصِّل Autodesk APS)، يخزّن المحلّل *عدد* العناصر — الجدران، البلاطات، الأعمدة، الكمرات، الأبواب، النوافذ، الفراغات، الطوابق — على سجل `bim-model`. وبصراحة منهجية: أعداد نُسخ لا مساحات مقيسة.
2. **الكمية (الحصر).** تحوّل دالة `deriveQuantitiesFromBim` تلك الأعداد إلى حصر تقريبي بأحجام اسمية موثّقة (≈12 م²/جدار، ≈120 م²/بلاطة، والأبواب/النوافذ بالعدد). وتُصنَّف وتُعلَّم بوضوح بأنها *مشتقة من الأعداد*.
3. **التكلفة (التسعير).** يُوزَّع معدل البناء الشامل لكل م² (معيار جدوى سيجما لنوع المشروع، مُعدَّلاً بمعامل الموقع) على عناصر التصنيف بحسب وزن نسبة تكلفة كل عنصر؛ أو في مسار BIM تُسعَّر كل كمية مشتقة بمعدّلها العنصري. ويحمل كل بند رمزه المعياري وقيمته ونسبته وثقته. ويُبرز أعلى ثلاثة عناصر كأهداف هندسة قيمة.
4. **جدول الكميات BOQ (المناقصة).** توليد BOQ مصنّف من النموذج (مع مراجعة قبل الحفظ)، والتحقق من BOQ قائم (القيمة = الكمية × السعر، الكميات/الأسعار غير الصفرية، شمول التصنيف، مطابقة الإجمالي)، ومقارنة عطاءات تُعلِّم الأسعار الشاذّة مقابل الوسيط.
5. **الحوكمة (المطابقة).** تقارن طبقة الحوكمة المصادر باستمرار وترفع نتائج حتمية: *انحراف الكمية* (BOQ مقابل المشتقة من BIM)، *انحراف التكلفة*، *الحصر الزائد*، *الكمية المكرّرة*، و*عدم تطابق الكمية مع التكلفة*. وتُزال التكرارات.

**أهم الميزات والأدوات (أربعة تبويبات + إجراء):**
- **تشغيل حوكمة الكميات** (الترويسة).
- **تبويب تقديرات التكلفة** — نموذج تقدير مصنّف؛ وكل تقدير قابل للتوسعة (الرمز، العنصر، الكمية، السعر، القيمة، النسبة).
- **تبويب إطار التصنيف** — مصفوفة مرجعية تربط كل عنصر برموز NRM/UniFormat/MasterFormat/CESMM.
- **تبويب التتبّع** — اختر البُعد والموضوع لعرض سلسلة دورة الحياة (BIM ← BOQ ← … ← نهائي) بقيمها وانحرافاتها ومُعتمِديها.
- **تبويب الحوكمة** — رسم خطورة + نتائج بقيمة مالية و**تمت المراجعة/استبعاد**.

**طريقة الاستخدام:** ارفع نموذج IFC (إن أردت مسار BIM) ← أنشئ تقديراً مصنّفاً أو ولّده من النموذج ← راجع إطار التصنيف ← **تشغيل حوكمة الكميات** ← افرز النتائج ← استخدم التتبّع.

**المُدخلات/المُخرجات:** تستهلك أعداد BIM وبيانات BOQ/خطة التكلفة؛ وتُنتج تقديرات مصنّفة وجداول كميات ونتائج حوكمة (بقيمة مالية) وسلاسل تتبّع. وتنتقل الكميات إلى المشتريات وتُغذّي الإيراد والتنبؤ.

**نصيحة:** الكميات المشتقة من BIM *إرشادية* — فتعامل مع انحراف الكمية كدعوة للتحقق من BOQ المقيس لا كدليل على خطئه.

### حوكمة المشتريات وسلسلة الإمداد — `/procurement`

**الوصول:** `canRunProcurement`.

**الغرض منها:** تخطيط حزم المشتريات، وتقييم المورّدين، وحوكمة سلسلة الإمداد عبر مطابقة الكميات والتواريخ (BIM مقابل المُشترى مقابل المُركّب، والمخطط مقابل الفعلي).

**أهم الميزات والأدوات:** **تشغيل حوكمة المشتريات**، وتبويب الحزم (مع أعمدة BIM/مُشترى/مُركّب)، وتبويب المورّدين (درجات تأهيل/أداء/مخاطر)، وتبويب الحوكمة (نتائج مع توصيات).

**طريقة الاستخدام:** أضِف حزماً (بكمية BIM وتاريخ التسليم) ← أضِف موردين ← **تشغيل حوكمة المشتريات** ← افرز النتائج.

**المُدخلات/المُخرجات:** كميات وتواريخ داخلة؛ درجات مورّدين ونتائج سلسلة إمداد خارجة. وتعود الفروق إلى حصر الكميات.

**نصيحة:** املأ *التسليم المخطط* و*طويل التوريد* — فهما أبكر إنذارات التأثير على البرنامج.

---

### حوكمة الإيرادات والتدفق النقدي — `/revenue`

**الوصول:** `canRunRevenueGovernance`.

**الغرض منها:** حوكمة ما *يُكتسَب*. تتتبّع دورة حياة الإيراد من التوقع حتى النهائي، وتُظهر كيف تحرّك الانحرافات NPV وIRR وفترة الاسترداد.

**أهم الميزات والأدوات:** **تشغيل حوكمة الإيراد**، وبطاقة الأثر الاستثماري (قبل←بعد)، وسلسلة حوكمة الإيراد مع نموذج **+ تسجيل مرحلة** (القيمة، المصدر، المُعتمِد، سبب التغيير).

**طريقة الاستخدام:** **+ تسجيل مرحلة** ← كرّر للقيم الفعلية ← اقرأ بطاقة الأثر ← **تشغيل حوكمة الإيراد**.

**المُدخلات/المُخرجات:** مراحل الإيراد ونموذج الجدوى داخلة؛ تحليل الأثر والسلسلة والنتائج خارجة. وترتبط بالتمويل والتنبؤ.

**نصيحة:** املأ *المصدر* و*المُعتمِد* و*سبب التغيير* — فهي ما يجعل السلسلة قابلة للتدقيق.

---

### حوكمة التمويل — `/funding`

**الوصول:** `canRunFunding`.

**الغرض منها:** حوكمة *كيفية* تمويل المشروع — التسهيلات والسحوبات ومراقبة DSCR والتعهّدات وخدمة الدين ومخاطر إعادة التمويل.

**أهم الميزات والأدوات:** **تشغيل حوكمة التمويل**، وبطاقة صحة التمويل، وبطاقة مركز التمويل، وجدول التسهيلات (مع خانة DSCR وعلامة *مخالفة*)، ونموذج **+ إضافة تسهيل**، ونتائج.

**طريقة الاستخدام:** **+ إضافة تسهيل** (المبالغ، تعهّد DSCR، DSCR الحالي، الاستحقاق) ← كرّر ← **تشغيل حوكمة التمويل** ← عالِج المخالفات.

**المُدخلات/المُخرجات:** التسهيلات (والإيراد) داخلة؛ درجة الصحة والإجماليات والنتائج خارجة. وتُغذّي DSCR القابليةَ للتمويل.

**نصيحة:** أدخِل *تعهّد DSCR* بجانب *DSCR الحالي* — فبدونه لا يظهر تجاوز.

---

### الحوكمة التنبؤية — `/predictive`

**الوصول:** `canRunPredictive`.

**الغرض منها:** نظام إنذار مبكر حتمي. خمسة توقّعات (تجاوز التكلفة، التأخير، فجوة الإيراد، مخاطر التوريد، مخاطر التمويل) مجمّعة في حالة واحدة (الأسوأ من الخمسة). وكل رقم من صيغة مُسمّاة.

**أهم الميزات والأدوات:** **تشغيل التوقعات**، وبطاقة الحالة المجمّعة، وخمس بطاقات توقّع (المؤشر، الخطورة، القيمة، *الأساس*، *الإجراء الموصى به*).

**طريقة الاستخدام:** **تشغيل التوقعات** ← اقرأ الحالة المجمّعة ثم كل بطاقة ← اقرأ *الأساس* واتّبع *الإجراء*.

**المُدخلات/المُخرجات:** حالة التكلفة/الجدول/الإيراد/المشتريات/التمويل داخلة؛ خمسة توقّعات وحالة مجمّعة خارجة.

**نصيحة:** «لا بيانات» تعني أن المؤشر المصدر لم يُسجَّل بعد — عبّئ الشاشة الكامنة وأعِد التشغيل.

---

### القابلية للتمويل البنكي — `/bankability`

**الوصول:** `canRunBankability`.

**الغرض منها:** تحويل بيانات الجدوى والتمويل إلى حزمة جاهزة للمُقرِض: DSCR مقابل التعهّد، وجدول إطفاء الدين، ومتطلبات التمويل، وحكم القابلية، وجاهزية حزمتي المستثمر والمُقرِض. وتقرأ البيانات القائمة بدل إعادة إدخالها.

**أهم الميزات والأدوات:** **تشغيل حوكمة القابلية للتمويل**، وبطاقة القابلية (حكم + مكوّنات)، وبطاقة متطلبات التمويل وDSCR، وقائمتا جاهزية المستثمر/المُقرِض، وجدول إطفاء الدين.

**طريقة الاستخدام:** تأكّد من وجود دراسة مستوى 2 وتسهيلات ← **تشغيل حوكمة القابلية** ← اقرأ الحكم والفجوة وDSCR ← أكمِل قوائم التحقق.

**المُدخلات/المُخرجات:** نموذج الجدوى والتسهيلات داخلة؛ درجة/حكم القابلية وجدول الدين وتحليل الفجوة والجاهزية خارجة — المُخرَج التجاري النهائي.

**نصيحة:** إن كان الحكم ضعيفاً، انظر أولاً إلى *فجوة التمويل* و*DSCR الفعّال*.

## 6. التسليم وحوكمة الموقع

ست شاشات حوكمة تعمل أثناء التنفيذ وعند الانتقال إلى التشغيل. تعمل كلٌّ بالطريقة نفسها: أضِف السجلّات، ثم **تشغيل الحوكمة**، واقرأ النتائج الحتمية، وافتح لوحة **تحليل الذكاء الاصطناعي**. وتُعرَض الحالة كمؤشّر ملوّن (أخضر/أصفر/برتقالي/أحمر).

---

### حوكمة السلامة والصحة المهنية — `/safety`

**الوصول:** `canRunSafety`.

**الغرض منها:** حوكمة الالتزام الفعلي بخطة السلامة المعتمدة في الموقع. وميزتها المميّزة: كل أمر إيقاف عمل يُتتبَّع عبر سلسلة (حدث سلامة ← إيقاف العمل ← تأخير ← المسار الحرج ← تمديد المدة ← جاهزية المطالبة).

**أهم الميزات والأدوات:** **تشغيل حوكمة السلامة**، وبطاقة درجات السلامة، وبطاقة الموقف، وبطاقة **سلسلة مطالبات إيقاف العمل**، وجدول السجلات، ونموذج **إضافة سجل** (مع مربّع **أمر إيقاف عمل** و**الأنشطة المتأثّرة** و**تمديد المدة**)، وسجل المخاطر.

**طريقة الاستخدام:** ابدأ بخطة السلامة ← لإيقاف: فعّل **أمر إيقاف عمل** وأدرِج الأنشطة المتأثّرة وتمديد المدة ← **تشغيل حوكمة السلامة** ← اقرأ سلسلة المطالبات (إصابة نشاط حرج تُحوّل الشارة إلى **المطالبة جاهزة**).

**المُدخلات/المُخرجات:** خطة السلامة ومفاتيح الأنشطة الحرجة داخلة؛ درجة الامتثال وسلسلة إيقاف العمل ← التأخير ← المسار الحرج ← تمديد المدة ← المطالبة، وسجل مخاطر خارج.

**نصيحة:** أدخِل دائماً مفاتيح الأنشطة المتأثّرة — فبدونها لا يُختبَر تأثير المسار الحرج وتبقى الشارة رمادية.

---

### حوكمة الحريق وسلامة الأرواح — `/fire-safety`

**الوصول:** `canRunFireLifeSafety`.

**الغرض منها:** حوكمة الامتثال لاستراتيجية الحريق واعتمادات الدفاع المدني. وتتعقّب الملاحظات المفتوحة ومخاطر مواعيد الاعتماد ودرجة جاهزية حريق.

**أهم الميزات والأدوات:** **تشغيل حوكمة الحريق والسلامة**، وبطاقة جاهزية الحريق، وبطاقة الموقف، وجدول السجلات، ونموذج **إضافة سجل**، وتجميع **الملاحظات المفتوحة**، ونتائج.

**طريقة الاستخدام:** **إضافة سجل** (النوع، الجهة، الحالة، الملاحظات، تاريخا التقديم والاعتماد المتوقّع) ← حدّث مع عودة الملاحظات ← **تشغيل الحوكمة** ← راقب تجميع الملاحظات (10+ بالأحمر) وأقرب اعتماد.

**المُدخلات/المُخرجات:** تقديمات الحريق وبيانات الدفاع المدني داخلة؛ درجة الجاهزية وتجميع الملاحظات ومخاطر الاعتماد خارجة.

**نصيحة:** حافِظ على **تاريخ الاعتماد المتوقّع** محدّثاً — فمؤشّرات التأخّر تُحتسَب منه مقارنةً باليوم.

---

### حوكمة الجهات — `/authority`

**الوصول:** `canRunAuthority`.

**الغرض منها:** سجل موحّد لكل تقديم واعتماد لدى الجهات (البلدية، الدفاع المدني، الكهرباء، المياه، الاتصالات، البيئة، الطرق، الصحة، وأخرى). ويحسب تلقائياً تعرّض المشروع للتأخير وأثره على المسار الحرج عند تجاوز الموافقة المتوقّعة تاريخها المطلوب.

**أهم الميزات والأدوات:** **تشغيل حوكمة الجهات**، وبطاقة جاهزية الجهات، وبطاقة **لوحة الموافقات وتعرّض التأخير** (مع جدول تأخير، صفوف المسار الحرج بالأحمر)، وجدول التقديمات، ونموذج **إضافة تقديم** (مع الأنشطة المتأثّرة وتاريخا المتوقّع والمطلوب).

**طريقة الاستخدام:** **إضافة تقديم** ← أدخِل المتوقّع والمطلوب والأنشطة المتأثّرة ← **تشغيل حوكمة الجهات** ← اقرأ لوحة التأخير.

**المُدخلات/المُخرجات:** التقديمات والتواريخ ومفاتيح الجدول داخلة؛ درجة الجاهزية وتعرّض التأخير وأثر المسار الحرج خارج — مغذٍّ مباشر لمطالبات تمديد المدة.

**نصيحة:** يعرض جدول التأخير فقط الصفوف التي تجاوز فيها المتوقّع المطلوب — فاملأ التاريخين ونشاطاً متأثّراً واحداً على الأقل.

---

### حوكمة المرافق — `/utility`

**الوصول:** `canRunUtility`.

**الغرض منها:** حوكمة جاهزية المرافق وحالة التوصيل (الكهرباء، المياه، الاتصالات، الغاز، الصرف، التبريد المركزي) مع تواريخ متوقّعة ومدى تأخير مقابل المطلوب.

**أهم الميزات والأدوات:** **تشغيل حوكمة المرافق**، وبطاقة الجاهزية، وبطاقة المركز، وجدول التوصيلات (مع أيام التأخير بالأحمر)، ونموذج **إضافة توصيلة**.

**طريقة الاستخدام:** **إضافة توصيلة** (النوع، الحالة، التواريخ) ← حدّث الحالة ← **تشغيل حوكمة المرافق** ← افحص عمود التأخير وبطاقتي المعرّض للخطر/أقصى تأخير.

**المُدخلات/المُخرجات:** تواريخ التقديم/المتوقّع/المطلوب داخلة؛ درجة الجاهزية وتعرّض التأخير خارج — شرط مسبق لبدء التشغيل.

**نصيحة:** المرفق على «لم يبدأ» دون تاريخ متوقّع يظهر «متعثّراً» — أضِف تاريخاً واقعياً فور التقديم.

---

### حوكمة الجاهزية التشغيلية — `/operational-readiness`

**الوصول:** `canRunOperationalReadiness`.

**الغرض منها:** حوكمة الانتقال من اكتمال الإنشاء إلى بدء التشغيل — أدلة التشغيل والصيانة، وسجلات الأصول، والتدريب، والاختبار والتشغيل، والتسليم، والتوظيف، وقطع الغيار، والضمانات — مجمّعة في درجة واحدة بدرجات فرعية (بدء التشغيل/التسليم/الاختبار والتشغيل).

**أهم الميزات والأدوات:** **تشغيل حوكمة الجاهزية**، وبطاقة الدرجة (مع الدرجات الفرعية)، وبطاقة المركز، وجدول العناصر، ونموذج **إضافة عنصر**.

**طريقة الاستخدام:** **إضافة عنصر** (الفئة، الحالة، نسبة الإنجاز، الاستحقاق) — عنصر لكل مُخرَج عبر كل الفئات ← حدّث ← **تشغيل الحوكمة** ← اقرأ النتائج (*معيق بدء التشغيل* = لا تُسلّم بعد).

**المُدخلات/المُخرجات:** عناصر التسليم/التشغيل والصيانة داخلة؛ درجة الجاهزية بدرجاتها الفرعية ونتائج المعيقات خارج — بوّابة إعلان الأصل جاهزاً.

**نصيحة:** الفئة الخالية تظهر *فجوة فئة* — تأكّد من وجود عنصر واحد على الأقل في كل فئة قبل بدء التشغيل.

---

### برنامج القبول — `/acceptance`

**الوصول:** `canEvaluateRules`.

**الغرض منها:** برنامج القبول الرسمي المكوّن من 23 اختباراً لإعلان جاهزية سيجما نفسها للإنتاج. يُنفَّذ كل اختبار مقابل الخدمات الحيّة ويُرجِع نجاح/فشل/متخطٍّ مع أدلّة. وهو مُشغِّل تحقّق من المنصّة لا سجل بيانات مشروع.

**أهم الميزات والأدوات:** **تشغيل كل الاختبارات**، وشريط ملخّص التشغيل، ومصفوفة الاختبارات الـ23، وتفصيل صف قابل للتوسعة (معيار النجاح، المدخلات، المخرجات، السبب، أدلّة JSON).

**طريقة الاستخدام:** يُحمَّل الكتالوج تلقائياً ← اقرأ المعايير ← **تشغيل كل الاختبارات** ← اقرأ الملخّص ← وسّع أي اختبار فاشل/متخطٍّ للسبب والأدلّة.

**المُدخلات/المُخرجات:** الحالة الحيّة لخدمات الحوكمة داخلة؛ تقرير نجاح/فشل/تخطٍّ مع أدلّة خارج — سجل القبول الرسمي.

**نصيحة:** «متخطّى» تعني غالباً نقص مُدخَل مطلوب للمشروع لا عطباً — هيّئ البيانات ثم أعِد التشغيل.

---
## 7. الأدوات والأدلة والتقارير

أدوات العمل التي تُحوِّل بها بيانات المشروع إلى مُخرَجات محكومة. والقاعدة ثابتة: يقترح الذكاء الاصطناعي، ويعتمد الإنسان، ويُسنَد كل رقم إلى مصدره.

---

### مُنشئ خط أساس البرنامج الزمني — `/baselines`

**الوصول:** التأليف يتطلّب `canSimulate`؛ والاعتماد يتطلّب `canApproveBaseline` (مسؤول، عميل، مالك، مجلس الحوكمة).

**الغرض منها:** يبني خبير تخطيط بالذكاء الاصطناعي ("planner-p6-25yr") خط أساس بأسلوب Primavera من نافذة مدّة العقد — WBS، ~90 نشاطاً، علاقات، طفو زمني، مسار حرج. ويُحجَز لمراجعة بتوقيعَين، ثم يُنزَّل ملفَّ `.xer` وتقرير PDF.

**أهم الميزات والأدوات:** بطاقة المشروع، وبطاقة **تحليل ضغط البرنامج** (تعجيل/مسار متوازي، بسقف 30%)، ونموذج توليد، وقائمة مهام قابلة للتوسعة، وإجراءات: **تنزيل ‎.xer**، **تقرير الجدول (PDF)**، **توقيع 1/2** ثم **توقيع 2/2 واعتماد**، **رفض**.

**طريقة الاستخدام:** تأكّد من المشروع ← (اختياري) **تحليل الجدول** ← أدخِل اسمك ← **توليد خط الأساس** ← وقّع 1/2 ثم 2/2 (بشخصَين مختلفَين) ← **تنزيل ‎.xer** أو **تقرير الجدول**.

**المُدخلات/المُخرجات:** تواريخ العقد (أو حزمة مخططات) داخلة؛ خط أساس معتمَد وملف `.xer` وتقرير PDF خارج.

**نصيحة:** يجب أن يصدر التوقيعان عن شخصَين *مختلفَين* — خطّط لمن يوقّع ثانياً.

---

### محاكاة السيناريوهات — `/simulation`

**الوصول:** `canSimulate`؛ واعتماد السيناريو يتطلّب `canEditPolicy`.

**الغرض منها:** بيئة اختبار آمنة لنسخ المشروع إلى سيناريوهات افتراضية دون المساس بالحالة المعتمَدة. راجع اللقطة أو ألغِها أو اعتمدها. كما تُسعِّر لوحةٌ تأخيراً مُفترَضاً عبر المحفظة.

**أهم الميزات والأدوات:** زر **نسخ سيناريو**، وبطاقات (مع **عرض الفرق** و**اعتماد**/**إلغاء**)، وتخطيط على مستوى المحفظة، ونموذج **تحليل ماذا-لو للتأخير**.

**طريقة الاستخدام:** **نسخ سيناريو** ← **عرض الفرق** ← **تحليل ماذا-لو** ← **اعتماد** (بصلاحية) أو **إلغاء**.

**المُدخلات/المُخرجات:** ينسخ ملخّص المشروع؛ ويُطلِق الاعتمادُ حدثَ `simulation.scenario.promoted`. وسيناريوهات التضارب تُطبَّق من `/clashes`.

**نصيحة:** ماذا-لو والفرق للاطّلاع فقط — استخدم **اعتماد** فقط حين تريد أن يصبح السيناريو الحالة المرجعية.

---

### سجل التضاربات — `/clashes` (و`/clashes/[id]`)

**الوصول:** العرض لأي مستخدم؛ والرفع `canIngest`؛ والاقتراح/المحاكاة `canEvaluateRules`؛ والاعتماد `canEditPolicy`.

**الغرض منها:** مراجعة تضاربات Navisworks/Revit. تقترح الشخصية "revit-clash-analyst" ثلاثة خيارات حلٍّ لكل تضارب؛ يختار المدير أحدها، ويحاكي الأثر، ويُصدر الاعتمادُ مراجعةَ جدول وخطابَ مطالبة FIDIC.

**أهم الميزات والأدوات:** منطقة رفع (**‎.xlsx/‎.xlsm**، ≤24 ميغابايت)، وشرائح تصفية، وبطاقات تضارب، وزر **اقتراح الخيارات** (ثلاثة خيارات)، و**محاكاة الأثر** ← **اعتماد**.

**طريقة الاستخدام:** **استيراد** الملف ← صفِّ (الحرجة فقط) ← **اقتراح الخيارات** ← اختر ← **محاكاة الأثر** ← **اعتماد**.

**المُدخلات/المُخرجات:** تصديرات تضاربات BIM داخلة؛ خيارات ومراجعة جدول وسيناريو وخطاب مطالبة (في `/letters`) خارج.

**نصيحة:** فُصِل "اقتراح" عن "رفع" عمداً — راجع القائمة أولاً لتصرف جهد الذكاء الاصطناعي على المهمّ.

---

### حِزَم المخططات — `/drawings`

**الوصول:** العرض لأي مستخدم؛ والرفع `canIngest`؛ و"توليد خط الأساس" `canSimulate`.

**الغرض منها:** إدخال مجموعات PDF ونماذج IFC. يؤرشف كل بايت (SHA-256)، ويستخرج مؤشّرات الطوابق والتخصّصات، ويسلّم الحزمة لمؤلِّف خط الأساس.

**أهم الميزات والأدوات:** منطقة رفع PDF (**‎.pdf**، ≤24 ميغابايت)، وبطاقات حِزَم (مع **توليد خط الأساس**)، وقسم **نماذج BIM (IFC)** (**‎.ifc**، ≤50 ميغابايت) ببطاقات عدّادات وفحوصات. (راجع التكاملات ← Autodesk BIM للتحويل السحابي الحيّ.)

**طريقة الاستخدام:** **إدخال** مجموعة PDF ← أكّد الطوابق والتخصّصات ← **توليد خط الأساس** ← لنموذج: ارفع `.ifc` ← **إدخال IFC**.

**المُدخلات/المُخرجات:** مخططات ونماذج داخلة؛ مؤشّرات تُغذّي خط الأساس؛ وتُراجَع التضاربات في `/clashes`؛ وتُغذّي الأعداد حصر الكميات.

**نصيحة:** تأكّد من طبقة نصية قابلة للقراءة في PDF — فالمستخرِج يقرأ النص لا الصور.

---

### خطابات FIDIC — `/letters`

**الوصول:** العرض لأي مستخدم؛ والصياغة `canEvaluateRules`؛ والاعتماد `canApproveLetter`. ولا يوجد **إرسال** عمداً.

**الغرض منها:** تصوغ خطابات FIDIC ثنائية اللغة عبر "fidic-redbook-expert" مع تذييل استشهاد إلزامي؛ ويتوقّف المسار عند "معتمَد" وتنزيل PDF.

**أهم الميزات والأدوات:** **صياغة رد على وارد** و**صياغة إشعار امتثال**، ومُنتقي نماذج، وشريط "يجب الرد قبل"، ومبدّل عربي/إنجليزي، ووسوم استشهادات، و**اعتماد** و**تنزيل PDF**.

**طريقة الاستخدام:** **صياغة رد/امتثال** ← أرسِل (مسودة مدعومة بمراجع) ← بدّل اللغة وتحقّق ← **اعتماد** ← **تنزيل PDF**.

**المُدخلات/المُخرجات:** خطاب وارد أو سرد امتثال داخل؛ خطاب معتمَد وPDF خارج. وتُنشئ اعتماداتُ التضاربات خطابات هنا.

**نصيحة:** يبقى تنزيل PDF مُعطَّلاً ما دام الخطاب مسودة — اعتمده أولاً.

---

### المصادر (المرجعية المعتمدة) — `/sources`

**الوصول:** أي مستخدم (للقراءة فقط).

**الغرض منها:** المرجعية المعتمدة للمعايير التي يُسمح للخبراء بالاستناد إليها. وأي استنتاج بلا وسم `[SOURCE: externalId]` يُعَدّ افتراضاً غير موثّق. وإليها تُحيل وسوم الاستشهاد.

**أهم الميزات والأدوات:** شرائح تصفية العائلة، وحقل بحث، وبطاقات مصادر (مع رابط **فتح المصدر**).

**طريقة الاستخدام:** صفِّ أو ابحث ← اقرأ النطاق ← **فتح المصدر**.

**المُدخلات/المُخرجات:** للقراءة فقط؛ تتحلّل وسوم الاستشهاد إلى هذه البنود.

**نصيحة:** عند التحقّق من خطاب أو تقرير، انقر وسم الاستشهاد ليقودك إلى المصدر ذاته.

---

### التقرير الشهري/الدوري — `/reports/monthly`

**الوصول:** العرض لأي مستخدم؛ ونموذج **التوليد** يتطلّب `canGenerateSummary`.

**الغرض منها:** يُولِّد تقريراً تنفيذياً مَرويّاً لفترةٍ ما عبر "report-narrator" — نص مترابط يبدأ بالحُكم ويُسنِد كل ادّعاء. ويُنزَّل PDF بالعربية والإنجليزية.

**أهم الميزات والأدوات:** نموذج التوليد (الدورية، الفترة، الجمهور، نوع السرد)، وقائمة تقارير، وعرض تفاصيل (رسوم + مقاييس + سرد + استشهادات)، و**طباعة** و**تنزيل PDF** عربي/إنجليزي.

**طريقة الاستخدام:** اختر الدورية والفترة والجمهور ونوع السرد ← **توليد** ← افتح الصف ← **PDF** أو **طباعة**.

**المُدخلات/المُخرجات:** جدول وتنبيهات وقرارات الفترة داخلة؛ تقرير مَرويّ وPDF عربي/إنجليزي خارج.

**نصيحة:** اختر *الجمهور* بعناية، وتحقّق من مؤشّر موثوقية البيانات قبل التعميم.

---

### مستودع الوثائق — `/repository`

**الوصول:** العرض لأي مستخدم؛ والتسجيل/التصنيف/OCR `canIngest`.

**الغرض منها:** مخزن سجلات الطبقة 1 (RFI، اعتمادات، NCR، طلبات تغيير، سجلات مشتريات/موارد/تكلفة، صور موقع، مراسلات، مستندات OCR). إضافةً فقط، موسوم تلقائياً، قابل للبحث.

**أهم الميزات والأدوات:** بحث، وشرائح نوع، ونموذج **تسجيل سجل** بثلاثة أوضاع (قياسي، مراسلة، **OCR** — صورة/PDF ≤20 ميغابايت)، وجدول سجلات مع **تصنيف**.

**طريقة الاستخدام:** ابحث/صفِّ ← **تسجيل سجل** ← للمسح: **OCR** ← **رفع وإجراء OCR** ← راجع النص ← **تصنيف**.

**المُدخلات/المُخرجات:** سجلات L1 ونص OCR داخلة؛ وسوم تلقائية تُغذّي البحث والحوكمة.

**نصيحة:** "بانتظار التفريغ اليدوي" تعني أن Claude Vision غير متصل — المسح مؤرشف؛ أعِد الرفع بعد تهيئة المفتاح.

---

### سلسلة الأدلة — `/evidence`

**الوصول:** أي مستخدم (للقراءة فقط).

**الغرض منها:** تعرض حزمة الأدلة الكاملة خلف تنبيه — التبرير، والمسار من الملف المصدر (SHA) إلى التنبيه، وتفصيل الثقة، والبيانات الخام. تجيب "لماذا أصدر النظام هذا وكم أثق به؟".

**أهم الميزات والأدوات:** قائمة تنبيهات، وبطاقة تبرير، وسلسلة مسار (الملف ← SHA-256)، وبطاقة ثقة (إجمالي + اكتمال/اتساق/موثوقية مصدر)، ومقتطفات خام.

**طريقة الاستخدام:** اختر تنبيهاً ← اقرأ التبرير وتتبّع المسار ← راجع تفصيل الثقة ← مرّر المقتطفات.

**المُدخلات/المُخرجات:** تقرأ التنبيهات وأدلّتها؛ بلا كتابة.

**نصيحة:** نسبة إجمالية مرتفعة مع اكتمال منخفض تعني ثقةً على بيانات جزئية — تحقّق قبل التصرّف.

---

### طابور الاعتماد — `/approval`

**الوصول:** `canEvaluateRules`.

**الغرض منها:** بوّابة الاعتماد البشري لقرارات الحوكمة. يعتمد المراجعون أو يرفضون أو يُقرّون؛ وتتطلّب القرارات الحرجة معتمِداً ثانياً مختلفاً (2 من 2)، وتُوسَم المتأخّرة بالتصعيد.

**أهم الميزات والأدوات:** بطاقات قرارات، وشريط حالة سلسلة (الحالة، "N/2 موافقات"، أسماء المعتمِدين، وسم **مُصعَّد**)، وثلاثة إجراءات: **اعتماد/رفض/إقرار**.

**طريقة الاستخدام:** راجع البطاقة ← **اعتماد/رفض/إقرار** ← للقرار الحرج يجب أن يعتمده مراجعٌ *مختلف* ثانياً.

**المُدخلات/المُخرجات:** أزواج تنبيه/قرار داخلة؛ إجراء مراجعتك يُكتَب على سلسلة الاعتماد.

**نصيحة:** لا يمكنك تقديم التوقيعَين على قرار حرج — هيّئ المراجع الثاني مسبقاً.

---

### مقارنة الذكاء الاصطناعي بالخبير البشري — `/comparison`

**الوصول:** العرض لأي مستخدم؛ والتسجيل `canEvaluateRules`؛ والحُكم `canEditPolicy`. ولا ذكاء اصطناعي في الصفحة.

**الغرض منها:** مقارنة جنباً إلى جنب بين مُخرَج الذكاء الاصطناعي والبشري لنفس المهمة. يسجّل المدير أيهما أقرب — وكل حكم مثال تدريبي للشخصيات.

**أهم الميزات والأدوات:** نموذج **تسجيل مقارنة**، وقائمة أزواج، وتفاصيل جنباً إلى جنب، وحقل ملاحظات تسوية، وثلاثة أزرار حُكم.

**طريقة الاستخدام:** **تسجيل مقارنة** ← افتح الزوج ← اكتب ملاحظات التسوية ← انقر الحُكم.

**المُدخلات/المُخرجات:** أزواج مُخرَجات داخلة؛ الحُكم يُغذّي تحسين الشخصية.

**نصيحة:** اكتب ملاحظات التسوية قبل الحُكم — فهي القيمة الحقيقية لحلقة التحسين.

## 8. الإدارة والصلاحيات

الأسطح التي تتحكّم بمن يدخل المنصّة وما يُسمح لكل دور به، وإعدادات الحوكمة والذكاء الاصطناعي على مستوى النظام.

---

### تسجيل الدخول — `/auth`

**الوصول:** متاح للجميع (بوابة الدخول).

**الغرض منها:** شاشة الدخول الوحيدة. وتؤدّي دور صفحة التهيئة الأولى عندما لا يوجد حساب مسؤول بعد.

**أهم الميزات والأدوات:** صفّ شارات الأدوار الـ15 (تملأ البيانات التجريبية)، وحقلا البريد وكلمة المرور (إظهار/إخفاء + تنبيه Caps Lock)، ومبدّلا اللغة والسمة، وشريط تهيئة (يعرض أمر `npm run user:create`).

**طريقة الاستخدام:** افتح `/auth` ← انقر شارة دورك أو اكتب بياناتك ← **تسجيل الدخول** ← عند ظهور شريط التهيئة نفّذ أمر CLI أولاً.

**نصيحة:** الكتابة في حقل البريد تُلغي الشارة المحدّدة — أكمِل الكتابة قبل الإرسال.

---

### حسابي — `/account`

**الوصول:** أي مستخدم مسجَّل.

**الغرض منها:** لقطة للقراءة فقط لجلستك (الاسم، الدور، نطاقات المشاريع، معاينة المفتاح)، ومنها تسجّل الخروج.

**أهم الميزات والأدوات:** بطاقة «مسجَّل الدخول»، ومعاينة المفتاح، و**تسجيل الخروج**، ورابط **عرض دليل الاستخدام**.

**طريقة الاستخدام:** افتح `/account` ← أكّد دورك ونطاقاتك ← **تسجيل الخروج**.

**نصيحة:** إذا «لم يرَ» زميل مشروعاً، فتحقّق من نطاقات مشاريعه هنا — فالحساب المحصور لا يرى غيرها.

---

### المساعدة — `/help`

**الوصول:** أي مستخدم.

**الغرض منها:** جولة موجزة عبر الأسطح الخمسة (الإدخال ← المراجعة ← الأدلة ← الاعتماد ← السياسات) وشرح وضع التهيئة والوصول حسب الدور.

**طريقة الاستخدام:** افتح `/help` ← اقرأ الحلقة الخماسية ← انقر أي خطوة للانتقال.

**نصيحة:** وجِّه أصحاب المصلحة الجدد إلى هنا أولاً.

---

### صلاحيات الأدوار — `/admin/roles`

**الوصول:** `canManageRoles` (مسؤول سيجما فقط).

**الغرض منها:** مصفوفة الصلاحيات في وقت التشغيل لتفعيل/تعطيل كل صلاحية لأيٍّ من الأدوار الـ15، تُطبَّق فوراً.

**أهم الميزات والأدوات:** جدول مصفوفة (مفتاح في كل خلية)، وعمود المسؤول مُقفل، وصفّا `canRead`/`canManageRoles` مُقفلان، وروابط استعادة و«إعادة الكل».

**طريقة الاستخدام:** حدّد الصفّ والعمود ← انقر المفتاح ← للتراجع استخدم المفتاح أو «استعادة».

**نصيحة:** تغيير الصلاحية يسري عند تحميل المستخدم التالي للصفحة — اطلب منه التحديث.

---

### إدارة المستخدمين — `/admin/users`

**الوصول:** العرض `canReadAll`؛ والإنشاء/التعديل/الحذف/المفاتيح `canManageRoles`.

**الغرض منها:** دليل حسابات أصحاب المصلحة — إضافة، تغيير أدوار ونطاقات، إعادة تعيين كلمات مرور، تدوير مفاتيح، حذف.

**أهم الميزات والأدوات:** جدول مستخدمين، ونموذج **مستخدم جديد**، وإجراءات لكل صفّ (تعديل، كلمة المرور، تدوير المفتاح، حذف).

**طريقة الاستخدام:** **مستخدم جديد** ← انسخ المفتاح فوراً (يُعرَض مرّة) ← **تعديل**/**كلمة المرور**/**تدوير المفتاح** ← **حذف** بتأكيد.

**نصيحة:** لا يمكن حذف المسؤول الوحيد النشط أو خفض رتبته — أنشئ مسؤولاً ثانياً أولاً.

---

### مركز إعداد الحوكمة — `/admin/governance`

**الوصول:** الصفحة `canEditPolicy`؛ وجدول الوكلاء قابل للتحرير بـ`canManageRoles` فقط.

**الغرض منها:** الشاشة الوحيدة لضبط محرّك الحوكمة الحتمي وأسطول الوكلاء معاً. وتسري في التشغيل التالي دون إعادة تشغيل.

**أهم الميزات والأدوات:** بطاقة المحرّك («التصعيد بعد (أيام)»، «التقييم التلقائي»، «الاعتماد المزدوج»، وثلاثة **أوزان تجميع** مجموعها 1)، وجدول الوكلاء (مفتاح **مُفعَّل** وقائمة **فئة النموذج**: الافتراضي/Haiku/Sonnet/Opus + حفظ لكل صفّ).

**طريقة الاستخدام:** اضبط نافذة التصعيد والمفاتيح ← عدّل الأوزان حتى 1.00 ← **حفظ إعداد الحوكمة** ← في جدول الوكلاء أوقِف وكيلاً أو ثبّت فئة نموذج ← **حفظ**.

**نصيحة:** ثبّت Opus فقط على الوكلاء الأهمّ حُكماً، واترك الباقي على الافتراضي لضبط التكلفة. (هذه رافعة ضبط تكلفة تكامل Claude.)

---

### محرِّر سياسة الحوكمة — `/admin/policy`

**الوصول:** `canEditPolicy`.

**الغرض منها:** عرض وتحرير سياسة الحوكمة — ربط FIDIC والمساءلة وطبقات التصعيد ومكتبة التدخّلات. وكل حفظ يُنشئ إصداراً جديداً.

**أهم الميزات والأدوات:** شارة الإصدار، وتبويبا «منظَّم»/«المحرِّر»، ومساحة نص JSON مع تحقّق حيّ، وزر **تنسيق**، وحفظ يتحوّل إلى «أصلِح JSON أولاً».

**طريقة الاستخدام:** اقرأ على «منظَّم» ← حرّر في «المحرِّر» ← **تنسيق** ← **حفظ** (إصدار جديد).

**نصيحة:** انقر **تنسيق** دائماً قبل الحفظ — فهو فحص تحليل نهائي.

---

### شخصيات الخبراء بالذكاء الاصطناعي — `/admin/personas`

**الوصول:** العرض لأي دور؛ والتحرير `canEditPersonas` (مسؤول فقط).

**الغرض منها:** سجل «أصوات الخبراء» خلف كل طبقة — مُوجِّهاتها وقواعدها وفئة نموذجها. وكل تعديل يُنشئ إصداراً جديداً.

**أهم الميزات والأدوات:** جدول شخصيات، وتصفية حسب الطبقة، ونافذة تعرض المُوجِّه والقواعد مع **تعديل** (للمسؤول) ← حفظ (إصدار جديد).

**طريقة الاستخدام:** صفِّ حسب الطبقة ← انقر شخصية ← (للمسؤول) **تعديل** ← **حفظ**.

**نصيحة:** الحفظ زيادةٌ في الإصدار — الإصدار السابق محفوظ ويمكن استعادته.

---

### سجل التدقيق — `/audit`

**الوصول:** `canReadAll`.

**الغرض منها:** سجل قابل للبحث ومختوم زمنياً لكل إجراء قرار — من اعتمد أو رفض أو أقرّ أي تنبيه، وتحت أي بند FIDIC ومستوى تصعيد.

**أهم الميزات والأدوات:** جدول حتى 300 إجراء (الوقت، الفاعل + تعليقه، الإجراء، الخطورة، رمز التنبيه، الطرف، بند FIDIC)، ومربع بحث، وأعمدة قابلة للفرز.

**طريقة الاستخدام:** ابحث بالفاعل/الرمز/الطرف/البند ← افرز ← اقرأ السطر الفرعي للتعليق.

**نصيحة:** ابحث برقم بند FIDIC لاستخراج كل قرار مرتبط به في عرض واحد.
